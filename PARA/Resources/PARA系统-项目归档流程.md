## 📋 项目完成后的标准流程

### 1. 确认项目完成
- [ ] 所有交付成果已完成
- [ ] README.md已更新（状态=已完成）
- [ ] 完成时间已记录

### 2. 项目总结
- [ ] 创建`项目总结.md`
- [ ] 记录成果统计
- [ ] 总结成功经验
- [ ] 分析问题和教训

### 3. 提取可复用资产
- [ ] 识别有价值的内容
- [ ] 提取到Resources/（工具、模板、方法）
- [ ] 提取到Areas/（经验、知识）

### 4. 归档项目
- [ ] 从Projects/移动到Archives/
- [ ] 更新Projects/列表
- [ ] 记录归档时间

### 5. 更新系统
- [ ] 更新MEMORY.md
- [ ] 更新心跳状态
- [ ] 清理待办任务

## 🎯 PARA系统使用规则

### Projects/（主动项目）
**定义**: 有明确目标、时间线、交付成果的项目
**何时**: 项目启动时创建
**何时完成**: 目标达成后归档

### Areas/（长期责任）
**定义**: 需要长期维护的领域
**内容**: 专业技能、责任范围、知识领域
**何时**: 提取项目经验时创建

### Resources/（主题参考）
**定义**: 可复用的资源、工具、模板
**内容**: 技巧库、模板、数据集
**何时**: 发现可复用内容时创建

### Archives/（已完成项目）
**定义**: 已完成的项目归档
**内容**: 项目总结、经验教训
**何时**: 项目完成后移动
