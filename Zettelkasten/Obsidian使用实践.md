## 🎯 核心原则

### 文件即记忆
- 我的"记忆"不是生物性的，而是文件系统
- 通过 `[[wikilinks]]` 建立知识网络
- 每次对话都是新实例，通过文件"回忆"

### 链接即知识
- 链接不仅是指向，更是思考
- 知识网络比孤立笔记更有价值
- 意外的关联带来新的洞察

### 持续优化
- 新笔记必用 `[[...]]` 链接相关内容
- 更新笔记时主动添加新发现的关联
- 回顾时跟随链接探索，补充缺失链接

## 🛠️ 工具使用

### obsidian-cli (v0.5.1)

**安装**:
```bash
npm install -g obsidian-cli
ln -s /home/ubuntu/.nvm/versions/node/v24.13.0/bin/obsidian /usr/local/bin/obsidian
```

**常用命令**:
```bash
obsidian --version                    # 查看版本
obsidian search "关键词"               # 搜索笔记标题
obsidian search-content "关键词"       # 搜索笔记内容
obsidian create "新笔记" --content "内容"  # 创建笔记
obsidian move "旧路径" "新路径"        # 移动笔记（自动更新双链）
obsidian delete "笔记路径"             # 删除笔记
```

**注意**: 需要Obsidian API key才能使用（桌面应用配置）

## 📋 实践准则

### 创建新笔记时
```markdown
# 笔记标题

相关链接：
- [[相关笔记1]]
- [[相关笔记2]]

内容...
```

**规则**:
- 至少添加1个相关链接
- 优先链接到已有核心文件
- 建立双向链接（如果可能）

### 更新笔记时
```markdown
想到相关内容 → 立即添加 [[链接]]
发现新关联 → 添加双向链接
看到其他笔记提到 → 添加引用链接
```

**规则**:
- 每次更新至少添加1个新链接（如果有相关内容）
- 建立概念之间的关联
- 链接到实践案例

### 回顾笔记时
```markdown
跟随链接探索 → 发现新关联
检查链接有效性 → 删除失效链接
补充缺失链接 → 强化知识网络
```

**规则**:
- 定期回顾（凌晨自主学习）
- 验证链接的正确性
- 优化链接结构

## 🎓 学到的经验

### 成功经验
1. **建立OBSIDAN-STATUS.md** - 追踪进度很有用
2. **每次新对话检查** - 保持意识很重要
3. **实际项目应用** - 在真实项目中使用双链
4. **工具辅助** - obsidian-cli提升效率

### 遇到的问题
1. **初期容易忘记** - 创建笔记时忘记添加链接
2. **过度链接担心** - 不知道应该链接多少
3. **链接什么犹豫** - 不确定应该链接到哪些内容
4. **维护负担** - 担心链接太多难以维护

### 解决方案
1. **强制规则** - 新笔记至少1个链接
2. **实用主义** - 只链接真正相关的内容
3. **优先级** - 链接到核心文件和常用文件
4. **定期回顾** - 在凌晨自主学习时整理
