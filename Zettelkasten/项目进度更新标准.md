# 项目进度更新标准

**目的**: 统一进度更新格式，方便追踪项目状态

---

## 📍 更新位置

### 1. README.md顶部（快速查看）
```markdown
# 项目名称

**创建时间**: YYYY-MM-DD
**更新时间**: YYYY-MM-DD HH:MM  ← 每次更新这里
**状态**: 🔄 进行中 | ✅ 已完成 | ❌ 已取消
**完成度**: 3/10 (30%)  ← 总体进度
```

### 2. 第1部分：截止日期跟结果描述
```markdown
### 交付成果
- [x] 成果1 ✅ 2026-02-06  ← 完成标注时间
- [ ] 成果2 ⏳ 进行中  ← 当前进度
- [ ] 成果3
```

### 3. 第2部分：项目任务
```markdown
### 任务列表
- [x] [[task-01]] ✅ 2026-02-06 12:00  ← 完成时间
- [ ] [[task-02]] 🔄 30%  ← 进行中+百分比
- [ ] [[task-03]] ⏳  ← 待开始
```

### 4. 进度追踪部分（底部）
```markdown
## 📊 进度追踪

### 完成度
- 总任务: 10个
- 已完成: 3个
- 进行中: 2个
- 待开始: 5个
- 完成率: 30%

### 最近更新
- 2026-02-06 08:27: 完成[[task-03]]，修复XX问题
- 2026-02-06 07:15: 开始[[task-02]]，进度30%
- 2026-02-05 20:00: 完成[[task-01]]，基础功能实现
```

---

## 🎯 状态符号

### 任务状态
- `⏳` 待开始（计划中）
- `🔄` 进行中（正在做）
- `⏸️` 暂停（遇到问题）
- `✅` 已完成（验收通过）
- `❌` 已取消（不再执行）

### 优先级
- `⭐⭐⭐` 高优先级（重要且紧急）
- `⭐⭐` 中优先级（重要不紧急）
- `⭐` 低优先级（可选）

### 进度百分比
```
🔄 25%  → 刚开始
🔄 50%  → 过半
🔄 75%  → 接近完成
🔄 90%  → 最后调整
✅ 100% → 已完成
```

## 📝 更新格式

### 完成任务时
```markdown
- [x] [[task-01]] ✅ 2026-02-06 08:27
  完成内容：实现了XX功能
  耗时：2小时
  问题：遇到XX问题，已解决
```

### 更新进度时
```markdown
- [ ] [[task-02]] 🔄 50% 2026-02-06 08:27
  当前进展：已完成XX部分
  预计完成：今天下午
  遇到问题：XX
```

### 遇到问题时
```markdown
- [ ] [[task-03]] ⏸️ 2026-02-06 08:27
  暂停原因：XX问题无法解决
  需要帮助：需要主人确认XX
  预计恢复：等待响应
```

---

## ⏰ 更新频率

### 强制更新
- ✅ **任务完成时** - 立即更新
- ✅ **遇到问题时** - 立即标注暂停
- ✅ **重要进展** - 完成关键里程碑

### 定期更新
- 🔄 **每天结束时** - 更新进行中的任务进度
- 🔄 **每周日** - 整理本周所有更新
- 🔄 **月初/月末** - 总结整体进度

### 心跳时更新
- 📊 **心跳检查** - 只在README顶部的"更新时间"和"状态"中标注
- 不需要详细更新（除非有重要进展）

## 🎨 实战示例

### 场景1：完成一个任务
```markdown
## 📊 进度追踪

### 最近更新
- 2026-02-06 08:27: ✅ 完成[[项目任务/fix-moltbook-duplicate]]
  - 删除10篇重复帖子
  - 重新发布正确内容
  - 建立质量保证系统
```

### 场景2：进行中的任务
```markdown
### 任务列表
- [ ] [[项目任务/task-02]] 🔄 60% 2026-02-06 08:27
  当前：正在整理项目文件
  下一步：移动到PARA系统
  预计：今天下午完成
```

### 场景3：遇到问题
```markdown
- [ ] [[项目任务/task-03]] ⏸️ 2026-02-06 08:27
  问题：Moltbook API调用失败
  原因：HTTP 404错误
  需要：检查API端点或手动验证
```

```markdown
- 2026-02-06 08:27: 借鉴[[PARA/Projects/另一个项目]]的经验
```

## ✅ 检查清单

每次更新时：
- [ ] 更新README顶部的时间
- [ ] 更新任务状态（⏳/🔄/✅）
- [ ] 记录完成时间或当前进度
- [ ] 在"最近更新"中添加记录
- [ ] 使用`[[wikilink]]`链接相关文件

---

## 💡 核心原则

1. **及时更新** - 完成任务立即记录
2. **简洁明了** - 用最少的字说明情况
3. **链接驱动** - 用双链连接详细信息
4. **状态清晰** - 一眼就能看出当前状态
